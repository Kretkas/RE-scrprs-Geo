# real_estate-scraper

## Что это
Парсит сайты недвижимости Грузии (MyHome, SS, Korter), выбирает последние квартиры за 24 часа и присылает в тг канал с фото и описанием.

## Цель
Стабильно работающий сервис, который ежедневно парсит MyHome, SS, Korter, находит новые объявления, получает номера телефонов владельцев, убирает дубликаты между сайтами и присылает результат в Telegram-канал.

## Стек / инструменты
- Язык: Python
- Основные библиотеки:
  - `scrapling` (для обхода защиты)
  - `requests` (для работы с Telegram API)
  - `json`, `re` (для обработки данных)
- Сервисы:
  - Telegram Bot API
- Инструменты:
  - `run_scrapers.sh` (для запуска и координации)
  - `cron` (для регулярного запуска `run_scrapers.sh`)

## Важные решения (уже принятые)
- **Использование `scrapling` для парсинга:** Вместо прямых запросов используется `scrapling`, так как целевые сайты (MyHome, SS.ge) имеют защиту от ботов. Этот инструмент повышает надежность, имитируя поведение браузера.
- **Хранение ID в `.txt` файлах:** ID уже просмотренных объявлений хранятся в простых текстовых файлах (`seen_ids.txt`). Этого достаточно для масштабов проекта и избавляет от необходимости поддерживать базу данных.
- **Прямая работа с Telegram API:** Вместо готовых библиотек используется `requests` для отправки сообщений. Это уменьшает количество зависимостей и дает полный контроль над логикой повторных попыток и обработкой ошибок, особенно лимитов Telegram.
- **Модульная структура скраперов:** Каждый сайт парсится отдельным файлом (`myhome_scraper.py`, `ss_scraper.py` и т.д.). Это позволяет легко отлаживать и изменять логику для одного сайта, не затрагивая остальные.

## Заметки
### Механизм отслеживания объявлений
- **MyHome.ge** и **SS.ge**: Для этих сайтов используются файлы `seen_myhome_ids.txt` и `seen_ss_ids.txt` для хранения ID уже опубликованных объявлений. Это необходимо, так как сайты постоянно обновляют объявления, меняя их дату на актуальную, и фильтрация по дате ненадежна.
- **Korter.ge**: Для этого сайта НЕ используется система `seen_ids`. Парсер полагается на точную дату **создания** объявления, которая доступна на сайте, и ищет только те, что появились за последние 24 часа. Это упрощает логику и избавляет от необходимости хранить ID.
